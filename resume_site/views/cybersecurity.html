Projects


Access management

The Access Management project is a self-directed initiative focused on identifying and mitigating the risks associated with a malicious actor having existing permissions within an organization's system. This project aims to conduct a comprehensive review of user permissions, assess potential vulnerabilities, and provide actionable recommendations to enhance security and prevent unauthorized access.

The Access Management project plays a crucial role in identifying and mitigating risks associated with unauthorized access within the organization. By conducting a thorough permissions review and providing actionable countermeasure recommendations, this project helps enhance the overall security posture, reduces the potential for data breaches, and strengthens the organization's resilience against malicious actors. The project contributes to a more secure and robust access control framework, safeguarding sensitive data and protecting the organization's reputation.


Data Leak Incident Documentation

The Data Leak Incident Documentation and Mitigation project focuses on analyzing and addressing a data leak incident caused by a customer success representative who gained access to an internal folder of documents containing sensitive information. The incident occurred when the representative inadvertently shared a link to the folder, resulting in the exposure of internal documents on a business partner's social media page. This project involves documenting the incident, assessing its impact, and proposing measures to mitigate future risks.

The Data Leak Incident Documentation and Mitigation project addresses a critical security incident, enabling the organization to learn from the incident and implement necessary measures to prevent future data leaks. By documenting the incident, conducting a root cause analysis, and proposing appropriate mitigations, this project helps the organization enhance its data protection practices, strengthen access controls, and improve user awareness and training. The project contributes to a more secure and resilient information handling framework, safeguarding sensitive data, and minimizing the risk of reputational damage.


Parking Lot USB

The USB Incident Investigation and Risk Analysis project focuses on examining the risks associated with a USB stick found in the parking lot of Rhetorical Hospital. The USB drive, bearing the hospital's logo, was picked up out of curiosity and brought back to the office for analysis. By conducting a comprehensive risk analysis, this project aims to identify potential security threats, data breaches, and vulnerabilities that may arise from the use of the USB stick.

The USB Incident Investigation and Risk Analysis project provides valuable insights into the risks associated with the use of USB devices within the hospital's network. By conducting a thorough risk assessment, analyzing potential malware threats, and evaluating data breach risks, this project helps raise awareness about the importance of secure USB usage and strengthens the hospital's security posture. The implementation of recommended risk mitigation measures and incident response protocols ensures a proactive approach to USB-related security incidents and safeguards sensitive patient information.



Incident Handler's Journal

The Incident Handler's Journal project focuses on creating a comprehensive journal that documents incident response activities, observations, and lessons learned during various cybersecurity incidents. As an incident handler, maintaining a detailed journal is essential for recording crucial information, tracking progress, and capturing key insights for future reference. This project aims to establish a structured and organized incident journal to facilitate effective incident response and enhance knowledge sharing within the cybersecurity team.

The Incident Handler's Journal project promotes effective incident response and knowledge sharing within the cybersecurity team. By maintaining a comprehensive journal, incident handlers can improve incident handling processes, foster collaboration, and enhance future incident response efforts.

Through this project, you develop a disciplined approach to incident documentation, facilitating the capture of critical information, timelines, and actions taken during incident response. By reflecting on each incident, you gain insights into incident patterns, emerging threats, and potential vulnerabilities within the organization's security infrastructure. This project contributes to building a repository of incident knowledge, enabling incident handlers to learn from past experiences and improve incident response capabilities.

The Incident Handler's Journal serves as a valuable resource for incident debriefings, regulatory compliance, and internal or external audits. It facilitates knowledge transfer, ensures consistency in incident documentation, and supports the continuous improvement of incident response processes.
